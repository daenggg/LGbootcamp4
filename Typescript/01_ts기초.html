Typescript 기초

JS와 비교하며 배워봅시다

타입스크립트 쓰는 이유

자바스크립트는 실행해야 오류를 알 수 있지만, 타입스크립트는 작성하는 순간 오류를 잡아줍니다.

Javascript - 실행 후 오류 발견
                        function greet(name) {
                            return name.toUpperCase()
                        }
                        greet('홍길동') // OK
                        greet(123) // 런타임 오류 -> 실행해야만 발견
Typescript - 작성 중 오류 감지
                        function greet(name : string) : string {
                            return name.toUpperCase()
                        }
                        greet('홍길동') // OK
                        greet(123) // 빨간 줄 -> 저장 전에 잡아줌

기본 타입 어노테이션

변수명 뒤에 ": 타입"을 붙여 어떤 값만 들어오는지 명시합니다.

                // 기본 타입: 변수명 : 타입 = 값
                const myName: string = '홍길동'
                const myAge: number = 25
                const isCoder: boolean = toUpperCase

                // 배열: 타입[]
                const scores: number[] = [85, 98, 78]
                const langs: string[] = ['html', 'css', 'js', 'ts']

                // Union: | 로 두 가지 이상 허용
                let userId: string | number = 'user-01'
                userId = 123 // 가능! 유니온이라서!

                // Optional: ? 로 undefined 허용
                let nickname?: string // 없어도 됨

Literal 타입 - 값 자체를 타입으로 쓰는 형태

string, number가 아닌 특정 값만 허용합니다. 오타나 잘못된 값이 들어오면 즉시 오류!

                // Literal 타입 : 딱 정해진 값만 허용
                let direction : 'left' | 'right' | 'up' | 'down' = 'left'
                direction = 'right' // OK
                // direction = "west" -> 오류! 허용되지 않은 값

                // 숫자 리터럴도 가능
                let dice: 1 | 2 | 3 | 4 | 5 | 6
                dice = // OK
                dice = 7 -> 오류

                // Union + Literal 조합 : 실전에서 상태/카테고리 제한에 자주 사용
                type Status = 'pending' | 'done' | 'cancelled'
                type Size = 'small' | 'medium' | 'large'

                let orderStatus: Status = 'pending'
                orderStatus = 'done' // OK
                // orderStatus = 'error' -> 오류! Status 없는 값!

                // type alias와 함께 쓰면 더 깔끔

type alias = 타입에 이름 붙이기

자주 쓰는 타입 조합에 이름을 붙여 재사용합니다.

                    // type 키워드로 이름 정의
                    type Grade = 'A' | 'B' | 'C' | 'F'
                    type Category = 'coffee' | 'tea' | 'dessert'
                    type UserId = string | number

                    // 이름만으로 타입 사용 가능
                    let grade: Grade = 'A'
                    let category: Category = 'coffee'
                    let id: UserId = 'user-01'

Interface - 객체의 설계도 (중요)

객체가 반드시 가져야 할 프로퍼티를 정의합니다. 빠진 프로퍼티가 있으면 즉시 오류!

                    // interface: 객체의 구조를 정의
                    interface Student {
                        id: number
                        name: string
                        score: number
                        email?: string // ? -> optional 없어도 됨
                        readonly enrolledAt: Date // readonly -> 수정 불가. 읽을 수만 있음 
                    }

                    // 설계도 대로 만들어야 함
                    const stu: Student = {
                        id: 1, name: '홍길동', score: 88,
                        enrolledAt: new Date('2024-03-01')
                    }

                    // interface를 함수 매개벼수 타입으로 활용
                    function getGrade(s: Student): string {
                        return s.score >= 90 ? 'A' : s.score >= 80 ? 'B' : 'C'
                    }

Interface extends - 상속으로 확장

공통 속성은 부모 interface에 한 번만 정의하고 extends로 물려받습니다.

                    // 부모 interface: 공통 속성
                    interface Person {
                        id: number
                        name: string
                    }

                    // extend: person 속성을 그대로 + 추가됩니다
                    interface Student extend Person {
                        grade: number // Person + grade
                    }

                    interface Teacher extends Persons {
                        subject : string // Person + subject
                    }

                    const stu: Student = {id : 1, name: '홍길동', grade: 2}
                    const tea: Teacher = {id : 2, name: '이순신', subject: '해군'}

함수 타입

매개변수와 반환값에 타입을 붙입니다. void 반환값이 없는 함수입니다.

                    // 기본: (매개변수: 타입): 반환타입
                    function add(a:number, b:number): number { 
                        return a + bs
                    }

                    // void: 반환값 없음
                    function log(msg: string): void {
                        console.log(msg)
                    }

                    // optional 매개변수 (?) / 기본값 (=)
                    function hello(name: string, title?: string): string {
                        return title ? `${title} ${name}` : name
                    }

                    function join(name: string, role: string = '학생'): string {
                        return `[${role} ${name}]`
                    }

클래스 + 접근 제어자

public / private / readonly로 각 프로퍼티의 접근 범위를 제한합니다.

                class BackAccount {
                    constructor(
                        public owner: string, // 클래스 외부에서 읽기/쓰기 가능
                        private balance: number, // 클래스 외부에서만 읽기/쓰기 가능
                        readonly backName: string // 클래스 외부에서 읽기 전용, 수정 불가 
                    ) {}

                    deposit(amount: number): void {
                        this.balance += amount
                    }

                    get currentBalance() : number {
                        return this.balance
                    }
                }

                const acc = new BackAccount('홍길동', 100000, '우리은행')
                acc.deposit(50000)
                // acc.balance = 0 -> 오류! private
                // acc.backName = 'X' -> 오류! readonly

제네릭 - 타입을 나중에 결정

T는 "타입 매개변수"입니다 함수나 인터페이스 정의할 때 타입을 미리 정하지 않고, 실제로 사용할 때 타입을 결정하여 코드의 재사용성을 극대화합니다. 

                // T는 어떤 타입이든 넣을 수 있는 '빈 박스'와 같습니다.
                // 태그가 뷰에 안 보인다면 <> -> < >
                function identify(value: T): T{
                    return value
                }

                // 호출할 때 타입을 결정 (string, number 등)
                const output1 = identify: <string> ('hello world') // T 가 string이 됨
                const output2 = identify: <number> (123) // T 가 number가 됨

                // 제너릭 interface : 데이터 구조는 같은데 내용물 타입만 다를 때 유용합니다.
                interface Box<T> {
                    content : T
                    addedAt: Date
                }

                const stringBox: Box<string> ={
                    content: '선물'
                    addedAt: new Date()
                }

                const numberBox: Box<number> ={
                    content: 123
                    addedAt: new Date()
                }
